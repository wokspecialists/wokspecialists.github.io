Docs

Quickstart, operating model, and troubleshooting for Chopsticks + Agents.

Quickstart

  1. Invite the main bot to your server.
  2. Invite N Agents. N equals the number of simultaneous voice channels you want.
  3. Run Lavalink and confirm host/port/password match your config.
  4. Start the main bot and Agent runners. Confirm logs show Agents connected.
  5. Join a voice channel, run play/queue commands, and validate audio output.

How multi-channel works

Each Agent is an independent bot user. In one server, one Agent can hold one voice connection at a time. If two voice channels need different audio simultaneously, deploy at least two Agents. More Agents equals more independent sessions pulled from the Agent pool.

Channel A: Agent 0001 plays Track A for Group A
Channel B: Agent 0002 plays Track B for Group B

Setup

  • Invite main bot + invite N Agents.
  • Run Lavalink. Confirm password/host/port match your environment.
  • Set tokens for Agents and confirm they connect.
  • Start bot and runners. Validate control messaging is live.

Troubleshooting

No audio

  • Confirm Lavalink is running and reachable.
  • Confirm host/port/password are correct.
  • Confirm your source/plugin setup matches your providers.
  • After Discord voice changes, re-test playback paths.

Agents offline

  • Confirm each Agent token is correct.
  • Confirm runner connectivity and control link is up.
  • Check logs for auth failures, reconnect loops, or rate limits.

Skip/stop behavior

  • Grace timing prevents thrash on rapid stop/skip.
  • If behavior feels delayed, check grace duration and queue rules.
