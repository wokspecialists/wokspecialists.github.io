Chopsticks Agents

Deploy extra Agents to run simultaneous sessions across multiple voice channels in the same server.

Why Agents exist

One Agent equals one concurrent voice session per server. If two channels need different audio at the same time, deploy two Agents. Add more Agents for more concurrent channels.

Example

Channel A: Agent 0001 plays Track A for Group A
Channel B: Agent 0002 plays Track B for Group B
More Agents equals more independent sessions.

Session control options
  • DJ-controlled: one user or role controls playback and queue
  • Open queue: channel members can add tracks (with configured limits)

Invite builder

Generate a batch of OAuth invites from the live Agent list. Invites are driven by data/agents.json.

Available Agents

Deployable now. If an invite is not configured, set clientId or inviteUrl in data/agents.json.

Coming Soon

Limits and notes

  • One voice connection per Agent per server
  • Multi-channel requires multiple Agents
  • Lavalink is required for audio
