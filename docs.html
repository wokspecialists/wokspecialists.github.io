Docs

Quickstart, operating model, and troubleshooting for Chopsticks + Agents.

Quickstart

  1. Invite the main Chopsticks bot to your server.
  2. Invite N Agents. N equals the number of simultaneous voice channels you want.
  3. Run Lavalink and confirm host/port/password match your config.
  4. Start Chopsticks and Agent runners. Confirm logs show Agents connected.
  5. Join a voice channel, run play/queue commands, and validate audio output.

How multi-channel works

Each Agent is an independent bot user. In one server, one Agent can hold one voice connection at a time. If two voice channels need different audio simultaneously, you deploy at least two Agents. More Agents equals more independent sessions, pulled from the Chopsticks Agent pool.

Setup

  • Invite main bot + invite N Agents.
  • Run Lavalink. Confirm password/host/port match your environment.
  • Set bot tokens for Agents and confirm they can connect.
  • Start main bot and Agent runners. Validate that control messaging is live.

FAQ

  • Do Agents work in private servers? Yes. Public or private is the same. They are deployable instances.
  • Do I need multiple Agents for one server? Only if you want simultaneous sessions in multiple voice channels.
  • Can one Agent serve multiple channels? Not simultaneously in the same server.

Troubleshooting

No audio

  • Confirm Lavalink is running and reachable from the bot host.
  • Confirm host/port/password are correct.
  • Confirm your source setup (plugins) matches your intended providers.
  • Voice server changes can break playback paths; re-test after Discord updates.

Agents offline

  • Confirm each Agent token is correct and has access to connect.
  • Confirm the control link between main bot and runners is connected.
  • Check runner logs for reconnect loops, auth failures, or rate limits.

Skip/stop behavior

  • Chopsticks uses a grace timer to avoid hard thrash when users stop/skip rapidly.
  • If behavior feels delayed, check the configured grace duration and queue rules.
